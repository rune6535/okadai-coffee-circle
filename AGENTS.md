# コーヒーサークル紹介Webサイト

## プロジェクト概要
大学のコーヒーサークルを紹介するWebサイトを作成する。
サークルの魅力を伝え、新規メンバーの獲得と活動情報の発信を目的とする。

## 技術スタック
- フロントエンド: HTML5, CSS3, JavaScript (バニラJS)
- スタイリング: Tailwind CSS
- レスポンシブデザイン: モバイルファースト
- ホスティング想定: GitHub Pages / Netlify

## サイト構成
### ページ
1. **トップページ (index.html)**
   - ヒーローセクション（メインビジュアル）
   - サークル紹介（概要）
   - 活動内容のハイライト
   - 新着情報・イベント告知
   - お問い合わせへのCTA

2. **活動内容ページ (activities.html)**
   - 定例会の紹介
   - カッピング（テイスティング）セッション
   - 焙煎体験
   - カフェ巡り
   - コーヒー勉強会

3. **メンバー紹介ページ (members.html)**
   - 代表・役員の紹介
   - メンバーの声
   - サークルの雰囲気が伝わる写真

4. **イベント・スケジュールページ (events.html)**
   - 年間スケジュール
   - 過去のイベント実績
   - イベントフォトギャラリー

5. **入会案内ページ (join.html)**
   - 入会方法
   - 会費・必要なもの
   - よくある質問（FAQ）
   - お問い合わせフォーム

## デザイン要件
### カラースキーム
- プライマリー: #6B4423 (コーヒーブラウン)
- セカンダリー: #D4A574 (ライトブラウン)
- アクセント: #C7A17A (ベージュ)
- 背景: #FAF8F3 (オフホワイト)
- テキスト: #2D2424 (ダークブラウン)

### フォント
- 見出し: 'Noto Sans JP', sans-serif (Bold)
- 本文: 'Noto Sans JP', sans-serif (Regular)
- 英語部分: 'Roboto', sans-serif

### デザインコンセプト
- 温かみのある、親しみやすいデザイン
- コーヒーの香りや雰囲気が感じられるビジュアル
- 清潔感とカジュアルさのバランス
- 初心者でも気軽に参加できる雰囲気

## 機能要件
### 必須機能
- レスポンシブナビゲーション（ハンバーガーメニュー）
- スムーススクロール
- 画像のフェードインアニメーション
- お問い合わせフォーム（基本的なバリデーション）
- SNSリンク（Instagram, Twitter, LINE）

### 推奨機能
- 画像ギャラリー（ライトボックス表示）
- アコーディオンメニュー（FAQ用）
- トップへ戻るボタン

## コンテンツガイドライン
### 画像
- プレースホルダー画像を使用（実際の画像は後で差し替え）
- 推奨サイズ: ヒーロー画像 1920x1080px、サムネイル 600x400px
- Alt属性を適切に設定

### トーン＆マナー
- フレンドリーで親しみやすい文章
- 専門用語は最小限に、初心者にも分かりやすく
- 「私たち」「一緒に」など、仲間意識を感じさせる表現

## 制約事項
- ブラウザ対応: Chrome, Firefox, Safari, Edge（最新版）
- モバイル対応: iOS Safari, Android Chrome
- 読み込み速度: 3秒以内（モバイル4G環境）
- アクセシビリティ: 基本的なWCAG 2.1 Level A準拠

## コーディング規約
### HTML
- セマンティックHTMLを使用
- 適切な見出しレベル（h1-h6）
- aria属性を必要に応じて追加

### CSS
- Tailwind CSSのユーティリティクラスを活用
- カスタムCSSは最小限に
- BEM記法（カスタムクラスを使用する場合）

### JavaScript
- ES6+の構文を使用
- コメントは日本語でも可
- 関数は小さく、再利用可能に

### ファイル構造
```
/
├── index.html
├── activities.html
├── members.html
├── events.html
├── join.html
├── css/
│   └── custom.css（必要に応じて）
├── js/
│   └── main.js
└── images/
    └── （プレースホルダー画像）
```

## 開発フェーズ
1. Phase 1: トップページの基本構造とデザイン
2. Phase 2: 各ページの作成とナビゲーション実装
3. Phase 3: インタラクティブ要素とアニメーション追加
4. Phase 4: レスポンシブ対応の最適化
5. Phase 5: 最終調整とテスト

## 注意事項
- 個人情報（メールアドレス、電話番号など）はダミーデータを使用
- 実在のサークルメンバーの写真は使用せず、プレースホルダーを使用
- お問い合わせフォームは見た目のみ（実際の送信機能は別途実装予定）
